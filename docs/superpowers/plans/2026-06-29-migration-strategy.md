# Migration Strategy Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace `prisma db push`-on-startup with `prisma migrate deploy` run from a container entrypoint script, so schema changes ship as migration files and apply non-destructively to existing databases.

**Architecture:** A `docker-entrypoint.sh` script runs `prisma migrate deploy` (idempotent — no-op if no pending migrations) before `exec`-ing into `node dist/main.js`. The `ensureSchema` method is removed from `PrismaService`. Dev workflow changes from `db push` to `migrate dev`.

**Tech Stack:** Prisma 6, SQLite, NestJS 11, Docker, shell.

## Global Constraints

- No legacy baselining — the project has no existing users (per user decision).
- SQLite only — `migration_lock.toml` locks the provider to `sqlite`.
- `prisma migrate deploy` is non-interactive and idempotent (Prisma's production-safe command).
- The entrypoint must `exec` into node so signals propagate correctly (SIGTERM for graceful shutdown).
- Filenames kebab-case per AGENTS.md.
- API is CommonJS (`strict: false`) — do not add `strict: true`.
- PrismaModule is `@Global()` — no need to import it into feature modules.
- The existing `COPY --from=builder /app/apps/api/prisma ./apps/api/prisma` in the Dockerfile already copies the whole prisma directory, so `migrations/` ships automatically once created.

### Task 2: Create the Docker entrypoint script

**Files:**
- Create: `apps/api/docker-entrypoint.sh`

**Interfaces:**
- Consumes: `apps/api/prisma/migrations/` (from Task 1), `apps/api/prisma/schema.prisma`.
- Produces: a script that Task 3 wires into the Dockerfile as `CMD`. It runs `prisma migrate deploy` then `exec`s into `node dist/main.js`.

- [ ] **Step 1: Create the entrypoint script**

Create `apps/api/docker-entrypoint.sh` with this exact content:

```sh
#!/bin/sh
set -e

echo "Running database migrations..."
npx prisma migrate deploy

echo "Starting TaskForge..."
exec node dist/main.js
```

The script:
- Uses `#!/bin/sh` (available on alpine, no bash dependency).
- `set -e` — exits on any command failure (migrate deploy failure stops the container before the app starts with a broken schema).
- `npx prisma migrate deploy` — Prisma's production-safe, non-interactive migration command. Idempotent: no-op if no pending migrations.
- `exec` — replaces the shell process with node so SIGTERM reaches the app for graceful shutdown.

- [ ] **Step 2: Make the script executable**

```bash
chmod +x apps/api/docker-entrypoint.sh
```

- [ ] **Step 3: Verify the script syntax**

```bash

```

Expected: no output, exit 0 (syntax check passes).

- [ ] **Step 4: Commit**

